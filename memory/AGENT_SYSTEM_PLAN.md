# نظام الوكلاء (مكاتب الحوالات) - خطة مستقبلية

> ⚠️ هذا النظام محفوظ للتنفيذ لاحقاً - يتطلب استشارة محامي أولاً

---

## الفكرة الأساسية

مكاتب الحوالات المالية (المنتشرة في سوريا) تعمل كنقاط شحن لمحفظة التطبيق.

```
العميل ← يذهب لمكتب حوالات ← يدفع نقداً ← المكتب يشحن رصيده في التطبيق
```

---

## الإعدادات المتفق عليها

| الإعداد | القيمة |
|---------|--------|
| دورة المحاسبة | 24 ساعة |
| مبلغ الضمان | يحدده المدير لكل وكيل |
| الحد الأقصى للشحن | لا يوجد |
| الحد الأدنى للشحن | لا يوجد |
| نسبة العمولة | يحددها المدير |

---

## آلية العمل

### من جهة العميل:
1. يفتح التطبيق ← يرى رقم هاتفه أو كود خاص
2. يذهب لأي مكتب حوالات
3. يعطيه الرقم/الكود + المبلغ النقدي
4. خلال ثوانٍ يظهر الرصيد في محفظته + إشعار

### من جهة الوكيل (مكتب الحوالات):
1. له لوحة تحكم خاصة (Agent Panel)
2. يدخل رقم/كود العميل + المبلغ
3. يضغط "شحن"
4. يأخذ النقد من العميل
5. الشحن يُخصم من رصيده المودع مسبقاً

### من جهة المدير:
1. يوافق على الوكلاء الجدد
2. يحدد مبلغ الضمان لكل وكيل
3. يحدد نسبة العمولة
4. يُحاسب الوكلاء كل 24 ساعة
5. يرى تقارير شاملة

---

## حلول المشاكل المحتملة

### المشكلة 1: الوكيل يأخذ النقد ولا يشحن
**الحل:** 
- إيصال فوري للعميل (SMS أو إشعار Push)
- العميل يتأكد قبل أن يغادر المكتب

### المشكلة 2: الوكيل يشحن لنفسه أو لأصدقائه
**الحل:**
- الوكيل يشحن من رصيده المودع مسبقاً فقط
- لا يستطيع الشحن أكثر من رصيده
- إذا انتهى رصيده، يجب أن يودع أولاً

### المشكلة 3: الوكيل لا يحاسب
**الحل:**
- مبلغ الضمان يغطي هذا
- إذا لم يحاسب خلال 24 ساعة ← يتوقف حسابه تلقائياً
- لا يستطيع الشحن حتى يحاسب

### المشكلة 4: نزاع على المبالغ
**الحل:**
- كل عملية مسجلة بالتاريخ والوقت بالثانية
- لا يوجد مجال للتلاعب - النظام يسجل كل شيء

## الشاشات المطلوبة

### 1. للعميل:
- عرض كود الشحن الخاص به
- رصيده الحالي
- قائمة نقاط الشحن القريبة (اختياري)
- سجل عمليات الشحن

### 2. للوكيل (Agent Panel):
- رصيده المتاح للشحن
- نموذج شحن رصيد عميل (رقم هاتف + مبلغ)
- سجل عملياته اليومية
- إجمالي الشحن اليومي
- عمولته المكتسبة

### 3. للمدير:
- قائمة كل الوكلاء
- إضافة وكيل جديد (مع تحديد الضمان والعمولة)
- تعديل/إيقاف/حذف وكيل
- رصيد كل وكيل
- تقارير الشحن اليومية
- نظام المحاسبة (كل 24 ساعة)
- إحصائيات شاملة

## الجداول المطلوبة في قاعدة البيانات

```
agents (الوكلاء):
- id
- name (اسم المكتب)
- owner_name (اسم صاحب المكتب)
- phone
- address
- city
- deposit_amount (مبلغ الضمان)
- commission_rate (نسبة العمولة)
- balance (الرصيد المتاح للشحن)
- status (active, suspended, pending)
- created_at
- approved_by

agent_topups (عمليات شحن الوكيل للعملاء):
- id
- agent_id
- customer_phone
- customer_id
- amount
- commission_earned
- created_at

agent_settlements (محاسبات الوكلاء):
- id
- agent_id
- period_start
- period_end
- total_topups
- total_commission
- net_amount (المبلغ المطلوب تحويله)
- status (pending, completed)
- settled_at
```

## APIs المطلوبة

### للوكيل:
- `POST /api/agent/topup` - شحن رصيد عميل
- `GET /api/agent/balance` - رصيدي المتاح
- `GET /api/agent/transactions` - عملياتي
- `GET /api/agent/stats` - إحصائياتي

### للمدير:
- `GET /api/admin/agents` - قائمة الوكلاء
- `POST /api/admin/agents` - إضافة وكيل
- `PUT /api/admin/agents/{id}` - تعديل وكيل
- `POST /api/admin/agents/{id}/suspend` - إيقاف وكيل
- `POST /api/admin/agents/{id}/activate` - تفعيل وكيل
- `POST /api/admin/agents/{id}/add-balance` - إضافة رصيد للوكيل
- `GET /api/admin/agents/settlements` - محاسبات الوكلاء
- `POST /api/admin/agents/{id}/settle` - تسوية حساب وكيل
