# تريند سورية - وثيقة متطلبات المنتج

## المشكلة الأصلية
بناء تطبيق تجارة إلكترونية متعدد البائعين باسم "تريند سورية" مع:
- سوق إلكتروني لجميع أنواع المنتجات
- توصيل مجاني داخل سوريا (مشروط)
- نظام متعدد البائعين مع وثائق رسمية
- تتبع الطلبات
- تقييمات ومراجعات المنتجات
- نظام أسئلة وأجوبة عامة للمنتجات
- دفع عبر شام كاش

## الجمهور المستهدف
- المشترون في سوريا
- البائعون الذين يريدون بيع منتجاتهم

## التقنيات المستخدمة
- **Backend**: FastAPI, Python
- **Frontend**: React.js, TailwindCSS, Framer Motion
- **Database**: MongoDB
- **Authentication**: JWT (Phone + Password)

- ✅ **شريط الشحن المجاني الذكي (تحسين UX)**

  - شريط تقدم (Progress Bar) يملأ تدريجياً
  - رسائل ذكية حسب الحالة:
    - "أضف X ل.س من نفس المتجر للتوصيل المجاني" (برتقالي)

    - "الشحن المجاني متاح فقط عند الشراء من متجر واحد" (أصفر)

    3. المجموع >= 150,000 ل.س

  - تكلفة الشحن: نفس المحافظة = 15,000 | قريبة = 15,000 | بعيدة = 25,000
